4. Como o FU vai sair organizado

Cada entidade alimenta uma fonte de FU diferente, mas todas aparecem no mesmo lugar — na tela Hoje e no Calendário Clínico (visualizações canônicas registradas em memória).

Mapa de FU por origem:
  • Lead → lead.proximo_followup (data) + pipeline_comercial_etapa (em qual estado está). Quando estado muda pra "Aguardando retorno" / "Retomar contato" / "Reaproximação", o sistema sugere data de FU.
  • Tutor → tutor.proximo_followup + estado_relacionamento (Em dia / Pet em tratamento / Precisa atenção / A recuperar / Inativo). Pra clientes "Precisa de atenção", FU é sugerido pelo sistema.
  • Pet → pet.proximo_followup + os 2 pipelines (clínico e fisio). Cada etapa pode disparar cadência automática.
  • Tudo isso já existe no Prisma — só preciso ligar os botões de avançar etapa + editar data ao backend.

Onde os FU aparecem:
  • Tela Hoje — bloco "Retornos vencidos" + "Próximos toques de cadência" + "Tutores a acompanhar".
  • Calendário Clínico — agendamentos com retorno + FU em formato calendário.
  • Inbox direito — pipeline visível pra ação imediata na conversa.
O que decidir antes de codar:
  • Toggle de 3 botões com cores distintas (turquesa / verde / índigo) → aprova?
  • WhatsApp Meta — se não permitir iframe (provável), faço nossa tela própria lendo da WhatsAppConversation. OK?
  • Pipeline COMERCIAL (Lead) com 10 etapas + barra + lista de pills clicáveis + FU date → aprova?
  • Pipelines CLÍNICO + FISIO (Pet) lado a lado com etapas clicáveis + retorno date + saldo pacote → aprova?
  • Estado de relacionamento do Tutor como badge (Pet em tratamento / Precisa atenção / etc) → aprova?
  • Avançar etapa via click direto (com confirmação) OU via dropdown? Sugiro click direto pra ser mais rápido.

Aprovando, PR fica com: 4 mudanças do v2 + pipelines + 3-way toggle + endpoint webhook resumo IA.
