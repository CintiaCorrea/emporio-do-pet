Inbox v2 — propostas pra você aprovar antes de codar

4 melhorias: toggle visível, cadastro inline, resumo IA do BotConversa, layout sem sobra.

1. Toggle visível pra alternar BotConversa ↔ Nativo

Substitui o link discreto atual ("Inbox nativo (preview)") por um switcher de duas pílulas centralizado, igual aos Settings de UI moderna. Dot verde mostra qual está ativo. Lembra que essa página é uma só (/dashboard/inbox) — só troca o conteúdo dentro.

3. Resumo da conversa (vindo via webhook do BotConversa)

Quando o BotConversa gera o resumo_ia da conversa, ele dispara um webhook. Hoje aponta pro Base44; só vamos mudar a URL pra apontar pro nosso endpoint. O resumo aparece bem em destaque no painel.

Como vai funcionar tecnicamente:
  • Criamos endpoint: POST https://app.emporiodopet.com.br/api/integrations/botconversa/webhook
  • Body esperado: { subscriber_id, phone, resumo_ia, atualizado_em } (mesma estrutura que já vai pro Base44 hoje)
  • O endpoint procura o Tutor/Lead pelo telefone e salva os campos resumo_ia + resumo_ia_atualizado_em que já existem no schema do Prisma.
  • No painel direito, sempre que tem resumo, aparece o card verde com selo "IA" e timestamp.
  • Botão "atualizar" recarrega do banco (não dispara webhook — webhook é controlado pelo BotConversa).
  • Você muda uma vez a URL do webhook no painel do BotConversa e fica funcionando.
